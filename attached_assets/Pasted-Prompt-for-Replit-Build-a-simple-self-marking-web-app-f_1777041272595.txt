Prompt for Replit

Build a simple self-marking web app for my son’s IGCSE Additional Mathematics 3-week crash course.

Purpose:
The student should be able to answer questions online, submit answers, receive immediate feedback, and view worked solutions.

App Name:
IGCSE Add Maths Crash Practice

Core Requirements:

1. Homepage
    * Title: IGCSE Add Maths Crash Practice
    * Short instruction:
        “Complete each section, submit your answers, and review the worked solutions immediately after checking.”
    * Topic cards:
        * Algebra Foundations
        * Quadratics
        * Differentiation
        * Trigonometry Survival
        * Mixed Exam Practice
2. Question Structure
    Each question must include:
    * Question number
    * Topic
    * Difficulty: Easy / Moderate / Exam Style
    * Question text
    * Input box for answer
    * Submit / Check Answer button
    * Instant result: Correct / Try Again / View Solution
    * Worked solution shown after submission
3. Answer Checking
    * Accept equivalent forms where possible, e.g.:
        * x=2 and 2
        * 3x^2 and 3*x^2
    * For now, use simple text normalisation:
        * remove spaces
        * convert to lowercase
        * allow multiple accepted answers
    * For quadratic solutions, accept answers in either order.
4. Progress Tracking
    * Show:
        * Total questions attempted
        * Correct answers
        * Percentage score
    * Store progress in browser localStorage.
    * Add a “Reset Progress” button.
5. Section Pages
    Each section should have at least 15 questions.
6. Worked Solutions
    After checking, show a collapsible worked solution box:
    * Step 1
    * Step 2
    * Final answer
7. Design
    * Clean student-friendly interface
    * Mobile responsive
    * Use soft colours
    * Avoid clutter
    * Add Sri Bintang Education small text/footer:
        “Prepared for internal learning support by Sri Bintang Education.”
8. Technical Preference
    * Use plain HTML, CSS, and JavaScript first.
    * No database needed at this stage.
    * Store all questions in a JavaScript array/object.
    * Keep the code easy for future expansion.
9. Important
    Do not create a login system yet.
    Do not use backend/database yet.
    Make it work fully in the browser.
